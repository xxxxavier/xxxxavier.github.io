Linked List Cycle

9月 24, 2018

It is a easy problem, however, the restrictive condition do not use extra space make this problem worth to be recorded.

Problem:

Given a linked list, determine if it has a cycle in it.

Follow up:
Can you solve it without using extra space?

No example.

Solution:

It is just like a two-man race, if there is a loop, the fast guy and slow always can meet at some point.

The question has a extented version, 142. Linked List Cycle II.

Problem:

Given a linked list, return the node where the cycle begins. If there is no cycle, return null.

No example.

Solution:

Basically, it is a mathematical problem. We can extend the previous function. After N round, run and walk meet, run have moved 2N steps, and 2N = X + aL + K, X is the length of link before the cycle, L is the length of the sycle, K is distence between the cycle begins and the meeting place. walk have moeved N steps, and N = X + bL + K. Then we can get X + K = (a-2b)L. It presents the relationship between X and K, and means when run and walk meet, the move k step can complete a cycle. Becasue of that, our solution is when run and walk meet, we let a another pointer start off from head, and after K step, it will meet, run or walk at the cycle begin. Even we does’t know the specific value of K, we go thought step by step, and when the two pointer is same, we get the result.
