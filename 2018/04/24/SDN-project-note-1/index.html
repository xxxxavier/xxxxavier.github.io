SDNproject

Project description

ECN part

This section will show the way to simulate ECN on Linux Virtual Machine.

Environment

Ubuntu 16.04.4

Tools

  • iperf - The ultimate speed test tool for TCP, UDP and SCTP
  • iftop - Used to listen to network traffic
  • tcpdump - Used to dump traffic on a network

Topology

Deployment

Router

In topology draft, Node 1 is a linux router. First, enable forwarding function.

Eable ECN function.

Node

Eable ECN function

By default, since , node0, node2 and node3 can not connect to each other. We need add a static route to eveynode. Take node0 as an example, add static route.

Now, we can use ping to test whether our configuration work.

Test

In test part, we use node3 as a server, node0 and node2 as client. Run two iperf server on node3.

Run iperf on node0 and use iftop to listen network interface and observe the change of sending rate on node0. To emulate a network burst, let nod2 send packet to node3 as well. Use tcpdump to capture the packet whose ECN bits is set to 11.

Result

Because of the congestion, the sending rate became lower than before.

The previous two tests were not congested, the third was.
