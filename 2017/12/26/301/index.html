301. Remove Invalid Parentheses

12月 26, 2017

今天更新的题目是301. Remove Invalid Parentheses，先贴题目



Remove the minimum number of invalid parentheses in order to make the input string valid. Return all possible results.

Note: The input string may contain letters other than the parentheses ( and ).


给出的例子是“()())()” -> [“()()()”,”(())()”]和“(a)())()” -> [“(a)()()”, “(a())()”]这道题目看起来跟之前的判定括号数量是否合法的题目有些相似，但是实际做起来却不相同。
对于括号的题目，我们首先应该想到的是使用堆（stack），遇到左括号就存入堆中，遇到右括号则出堆。在这道题中我们使用一个变量count来模拟堆操作。首先我们从左向右处理输入的字符串，此时我们处理的是“）”，如果有多余的“）”，会尝试将前面出现的“）”符号其中人一个删除，看得到的结果是否符合要求，这里需要注意的一点是如果原字符串是“））”，这种情况下删除第一个和删除第二个是同样的解。从左向右处理完之后将字符串反转，此时需要处理的是“（”，通过对每个可能的多余符号进行remove操作，当左右括号都满足要求是，该答案就是我们要的解。

最近刷题的感受是本科的基础太薄弱了，当时在本科的时候对ACM有些不屑，觉得这个东西跟奥数有些相像，但是这种东西在实际生产中能排上的用处非常少，但是在实际刷题的过程中，我发现做题能帮你更好的理解一些知识点以及帮你补上一些漏洞，C++是我很早之前学的东西，堆栈什么的也是在最早就有接触到的概念，但是不论是堆栈还是map这些东西在之前的开发还是作业中都很少会用到，做题能给我们创造一个场景去使用。
