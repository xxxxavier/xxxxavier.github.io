146. LRU Cache

4月 08, 2019

Leetcode第146题，LRU Cache， 一道非常经典的题目。

Design and implement a data structure for Least Recently Used (LRU) cache. It should support the following operations: get and put.

get(key) - Get the value (will always be positive) of the key if the key exists in the cache, otherwise return -1.
put(key, value) - Set or insert the value if the key is not already present. When the cache reached its capacity, it should invalidate the least recently used item before inserting a new item.

Follow up:
Could you do both operations in O(1) time complexity?

Example:

LRUCache cache = new LRUCache( 2 / capacity / );

cache.put(1, 1);
cache.put(2, 2);
cache.get(1); // returns 1
cache.put(3, 3); // evicts key 2
cache.get(2); // returns -1 (not found)
cache.put(4, 4); // evicts key 1
cache.get(1); // returns -1 (not found)
cache.get(3); // returns 3
cache.get(4); // returns 4

LRU Cache这个题目在Leetcode里算是经典中的经典了，一直于这道题出了这么久了还会在面试中出现。其实设计一个LRU cache的数据结构本身不是特别难，但是将他限制到所有操作都是O(1)的时间复杂度，就比较麻烦，这道题的考察点主要在于hash和linklist。

第一次用Java做这一道题，才知道java本身并没有一个内建的double linked list，因此需要自己动手（cpp万岁）。

这一题的具体思路是利用linked list的特性，我们将最近访问的值永远放置在linked list的头部，因此当linked list的大小超出我们初始时设置的capacity时，我们可以得出，位于linked list最后的值一定是最不活跃的或者说最少被访问的值，因此我们要将他剔除。另外我们需要一个map，因为linked list的特性是无法根据下标访问，因此我们要满足查询操作也达到O（1）的时间复杂度，需要建立一个key和linked list node的关系。

另外给自己一个note，之前一个电面，15分钟，挂了，不太明白为什么，今天明白了，因为就问了4个tech的问题，所以其中一个没答好就不太可能进入下一轮。其中一个问题是Java中Set和List的区别。这个问题，我非常干脆地把Java的List和C++ STL中的list联系到了一起。但是实际上Java中的list就是array，而C++中的list是Linked List，我还在答案中特地强调了，list是不能根据index访问的，这是大错特错的，因此没有进第二轮也非常的正常，非常的遗憾。
